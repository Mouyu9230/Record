**树（Tree）的基本概念与定义**

1. **树**
   由有限个结点组成的层次结构。若不为空，有且仅有一个**根结点**，其余结点分成若干互不相交的**子树**。

2. **结点（Node）**
   树中的基本元素。

3. **根结点（Root）**
   树的起始结点，没有父结点。

4. **父结点 / 子结点**
   直接相连的上层结点为父结点，下层为子结点。

5. **兄弟结点**
   具有同一父结点的结点。

6. **度（Degree）**

   * 结点的度：该结点子结点的个数
   * 树的度：树中所有结点度的最大值

7. **叶子结点（Leaf）**
   度为 0 的结点。

8. **分支结点 / 内部结点**
   度不为 0 的结点（非叶子）。

9. **层（Level） / 深度**

   * 根结点层数通常为 1（有时为 0）
   * 结点所在的层号

10. **树的高度（Height） / 深度（Depth）**
    树中结点的最大层数。

11. **子树（Subtree）**
    某结点及其所有后代结点构成的树。

12. **森林（Forest）**
    多棵互不相交的树的集合。

13. **有序树 / 无序树**

    * 有序树：子结点有先后次序
    * 无序树：子结点无顺序

14. **路径（Path）**
    从一个结点到另一个结点经过的结点序列。

15. **路径长度**
    路径中边的数量。

16. **祖先 / 后代**
    路径上位于上方的为祖先，下方的为后代。

**树的存储结构（3 种常见表示法）**

---

## 1. 双亲表示法（Parent Representation）

**思想**
每个结点只记录它的父结点下标。

**结构定义**

```c
#define MAXN 100
typedef struct {
    char data;
    int parent;   // 父结点下标，根结点为 -1
} PNode;

PNode tree[MAXN];
```

**特点**

* 易于找父结点
* 不便于找孩子结点
* 适合“由下向上”操作

---

## 2. 孩子表示法（Children Representation）

**思想**
每个结点保存所有孩子的链表。

**结构定义**

```c
typedef struct ChildNode {
    int child;                // 孩子结点下标
    struct ChildNode* next;
} ChildNode;

typedef struct {
    char data;
    ChildNode* firstChild;    // 孩子链表头指针
} CNode;

CNode tree[MAXN];
```

**特点**

* 易于找孩子
* 不便于找父结点
* 适合“由上向下”遍历

---

## 3. 孩子-兄弟表示法（Child-Sibling Representation）

**思想**
用二叉链表表示树：

* 左指针 → 第一个孩子
* 右指针 → 下一个兄弟

**结构定义**

```c
typedef struct CSNode {
    char data;
    struct CSNode* firstChild;
    struct CSNode* nextSibling;
} CSNode;
```

**特点**

* 既能找孩子，又能找兄弟
* 结构统一，节省空间
* 任何树都可转为二叉链表形式

---

## 三种表示法对比

| 表示法   | 找父 | 找孩子 | 空间 | 适用场景 |
| ----- | -- | --- | -- | ---- |
| 双亲    | 快  | 慢   | 最省 | 查祖先多 |
| 孩子    | 慢  | 快   | 较大 | 遍历子树 |
| 孩子-兄弟 | 一般 | 一般  | 较省 | 通用表示 |
